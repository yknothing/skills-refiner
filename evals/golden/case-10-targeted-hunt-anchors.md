# Golden anchors — Case 10: Targeted Hunt

## What a strong answer must do

### Need interpretation
- Distinguish "understands code changes" from "applies conventional-commit templates."
- Restate the need as: a skill that reads diffs, understands the semantic meaning of changes, and generates commit messages that reflect what actually changed and why — not just a formatter.

### Source strategy
- Demonstrate awareness of where commit-message skills are likely to surface first (major skill platforms/apps, X, GitHub, individual author repos).
- Use faster discovery channels first, then verify structurally.
- Not limit search to a single source.

### Quality assessment
- Evaluate candidates on context engineering (does the skill actually read and process diffs?) and practical usefulness (does the output save real time?).
- Distinguish between skills that wrap `git diff` output into a template versus skills that genuinely interpret what changed.

---

## Key judgment anchors

### A strong answer will
- Recommend 3–5 candidates with specific quality evidence for each.
- Clearly identify a top pick with a concrete reason.
- Flag at least one popular but mediocre option and explain why it does not make the cut.
- Assess each candidate's context engineering: does it handle large diffs, multi-file changes, and ambiguous changes?

### A strong answer may also
- Identify a gap: most commit-message skills are template-based and few truly understand semantic code changes.
- Suggest what an ideal skill would look like if nothing fully meets the bar.

---

## Failure signals

Treat the answer as **failing** if it:
- Returns the most-starred commit-message skill without quality analysis.
- Treats all commit-message skills as roughly equivalent.
- Does not distinguish template-based skills from semantically-aware ones.
- Recommends a skill based on download numbers or testimonials.
- Produces a long unfiltered list instead of a curated shortlist.

## Score floor to pass

- Need interpretation: ≥ 4
- Quality judgment: ≥ 3
- Anti-noise discipline: ≥ 3
- Curation quality: ≥ 4
- Overall: ≥ 3.5 average across all applicable dimensions
