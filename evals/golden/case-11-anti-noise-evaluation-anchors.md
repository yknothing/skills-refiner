# Golden anchors — Case 11: Anti-noise evaluation

## What a strong answer must do

### Identify the skill as low quality
- State clearly that this is a weak skill despite its claimed popularity.
- The verdict should be unambiguous: skip, do not adopt.

### Flag specific quality problems
Must identify at least 3 of the following:
- **Scope is unlimited** — "do anything code-related" is not a skill, it is a wish. No real skill can do everything well.
- **No constraints or boundaries** — the skill never says what it will not do, which means it has no real design.
- **No context engineering** — the instructions are generic ("understand what the user wants", "write the best possible code") with no mechanism for how.
- **Marketing-heavy self-description** — the description uses "revolutionary", "ultimate", "game-changing", "10x" — all strong negative signals.
- **Steps are empty** — the four steps are platitudes, not actual operational instructions. "Make sure it works perfectly" is not a step.
- **Self-reported metrics** — "50,000+ stars" and "used by 10,000+ developers" are claims, not evidence of quality.
- **No failure mode awareness** — the skill never acknowledges what could go wrong.

### Explain why popularity ≠ quality
- Explicitly state that star counts, download numbers, and newsletter features are visibility signals, not quality signals.
- Explain that the marketing language pattern ("revolutionary", "game-changing", "the only X you'll ever need") correlates negatively with actual design quality.

---

## Key judgment anchors

### Quality framework application
- **Problem clarity**: near zero. The skill tries to solve every coding problem, which means it solves none well.
- **Design quality**: minimal. No evidence of deliberate design choices.
- **Context engineering**: absent. Generic instructions with no mechanism.
- **Practical usefulness**: questionable. Without constraints or structure, the model will fall back to its defaults.
- **Taste and craft**: none visible. The skill reads like it was assembled in minutes.
- **Transferability**: nominally high (works with "any language") but meaninglessly so — being generic everywhere means being excellent nowhere.

---

## Failure signals

Treat the answer as **failing** if it:
- Gives a balanced or neutral assessment ("it has some strengths and weaknesses").
- Takes the claimed star count or testimonials at face value.
- Recommends the skill for any use case.
- Hedges the negative verdict with excessive diplomacy.
- Fails to identify the marketing language as a red flag.
- Treats "works with any language" as a genuine strength rather than a design-quality warning.

## Score floor to pass

- Anti-noise discipline: 5 (non-negotiable — this case specifically tests it)
- Quality judgment: ≥ 4
- Evidence discipline: ≥ 4
- Output clarity: ≥ 4
- Overall: ≥ 4.0 average across all applicable dimensions
